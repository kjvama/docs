Document Title

Knowledge Transfer Principles

==================================================
Purpose
==================================================
프로젝트 전체에서 일관된 설계와 의사결정을 유지하기 위한 지식전수 원칙을 정의한다.

==================================================
Background
==================================================
개발은 시간이 지나면서 작성자의 기억에 의존하면 유지보수가 어려워진다.
따라서 프로젝트의 설계 의도와 결정 이유를 문서로 남긴다.

==================================================
Scope
==================================================
This document applies to:
- UI Design
- Database Design
- API Design
- Documentation
- Naming Conventions

==================================================
Principles
==================================================
The KJVAMA project adopts the following knowledge transfer principles.

본 King James Version Bible Ask-Me-Anything Project (이하 KJVAMA)는 다음과 같은 지식전수의 원칙을 가진다.
- 코드는 기능을 만든다.
- 문서는 의도를 남긴다.
- 설계 결정은 이유와 함께 기록한다.
- 컴포넌트는 교체 가능하게 만든다.
- 새로운 개발자는 문서만 보고도 전체 흐름을 이해할 수 있어야 한다.

==================================================
Decision
==================================================
Knowledge Transfer Principles를 프로젝트의 최상위 원칙으로 채택한다.
