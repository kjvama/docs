This document defines the knowledge transfer principles that ensure consistent design and decision-making throughout the project.

==================================================
Background
==================================================
English Version
As a project grows, relying on memory alone makes long-term maintenance difficult.
To ensure continuity, the project's design intent and the reasons behind key decisions must be documented.

Korean Version
개발은 시간이 지나면서 작성자의 기억에 의존하면 유지보수가 어려워진다.
프로젝트의 연속성을 유지하기 위해 설계 의도와 주요 설계 결정의 이유를 문서로 남긴다.

==================================================
Scope
==================================================
This document applies to all design and development activities in the KJVAMA project.

- UI Design
- Database Design
- API Design
- Documentation
- Naming Conventions

==================================================
Principles
==================================================
English Version
The KJVAMA project adopts the following knowledge transfer principles.

1. Code creates functionality.
2. Documentation preserves intent.
3. Record design decisions together with their reasons.
4. Components should be designed to be replaceable.
5. A new developer should be able to understand the overall project by reading the documentation.

Korean Version
본 King James Version Bible Ask-Me-Anything Project (이하 KJVAMA)는 다음과 같은 지식전수의 원칙을 가진다.

1. 코드는 기능을 만든다.
2. 문서는 의도를 남긴다.
3. 설계 결정은 이유와 함께 기록한다.
4. 컴포넌트는 교체 가능하게 만든다.
5. 새로운 개발자는 문서만 보고도 전체 흐름을 이해할 수 있어야 한다.
